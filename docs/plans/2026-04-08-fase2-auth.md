# Fase 2 — Auth Implementation Plan

**Data:** 2026-04-08
**Goal:** Sign in, sign up, sign out, token renewal, and route protection.

---

## Architecture

- `accessToken` mem only (`tokenStore`) — no localStorage, no JS-accessible cookie
- `refreshToken` httpOnly cookie — server only via `features/auth/server.ts`
- `apiClient` 401 → calls `renewTokenCallback` → retry once → fail = clear token
- Auth routes redirect `/` via `beforeLoad` if no session
- Session restore on client: root `beforeLoad` calls `getInitialToken` server fn

## ✅ Task 4: Server functions — `features/auth/server.ts`

| Function | Endpoint | Cookie |
|---|---|---|
| `serverSignUp` | POST `/v1/auth/signup` | — |
| `serverSignIn` | POST `/v1/auth/signin` | sets `vid_rt` |
| `serverSignOut` | POST `/v1/auth/signout` | deletes `vid_rt` |
| `renewToken` | POST `/v1/auth/renew-token` | updates `vid_rt` |
| `getInitialToken` | POST `/v1/auth/renew-token` | updates `vid_rt` |

Cookie: `vid_rt` (httpOnly, sameSite=lax, 30d)
