# Fase 2 — Auth Implementation Plan

**Data:** 2026-04-08
**Goal:** Sign in, sign up, sign out, token renewal, and route protection.

---

## Architecture

- `accessToken` in memory only (`tokenStore`) — never localStorage, never a cookie accessible to JS
- `refreshToken` in httpOnly cookie — set/read exclusively by server functions in `features/auth/server.ts`
- `apiClient` intercepts 401 → calls `renewTokenCallback` → retries once → on failure clears token
- Authenticated routes redirect to `/` via `beforeLoad` if no session
- Session restored on client via root route `beforeLoad` calling `getInitialToken` server function

## ✅ Task 4: Server functions — `features/auth/server.ts`

| Function | Endpoint | Cookie |
|---|---|---|
| `serverSignUp` | POST `/v1/auth/signup` | — |
| `serverSignIn` | POST `/v1/auth/signin` | sets `vid_rt` |
| `serverSignOut` | POST `/v1/auth/signout` | deletes `vid_rt` |
| `renewToken` | POST `/v1/auth/renew-token` | updates `vid_rt` |
| `getInitialToken` | POST `/v1/auth/renew-token` | updates `vid_rt` |

Cookie name: `vid_rt` (httpOnly, sameSite=lax, 30 days)
